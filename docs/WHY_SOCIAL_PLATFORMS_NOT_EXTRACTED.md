# Why "All Social Platforms" is NOT Extracting Social Platform Nodes

## Problem Analysis

### User Prompt:
```
"Generate AI content daily and post automatically on all social platforms"
```

### Expected Behavior:
- Should extract: `instagram`, `twitter`, `linkedin`, `facebook` (all social platform nodes)
- Should include these in variations

### Actual Behavior:
- Variations show: `google_sheets`, `google_gmail`, `ai_chat_model`, `manual_trigger`
- **NO social platform nodes** (instagram, twitter, linkedin, facebook)

## Root Cause Analysis

### Issue 1: Filter is Too Strict

**Location:** `filterToExplicitlyMentionedNodes()` (line 1633)

**Problem:**
```typescript
// Direct mention: "instagram" in prompt → instagram node
if (promptLower.includes(nodeTypeLower)) {
  explicitlyMentioned.add(nodeType);
}
```

**Why it fails:**
- User says: "all social platforms"
- Filter checks: Does prompt include "instagram"? → NO
- Filter checks: Does prompt include "twitter"? → NO
- Filter checks: Does prompt include "linkedin"? → NO
- Filter checks: Does prompt include "facebook"? → NO
- **Result:** All social platform nodes are FILTERED OUT

### Issue 2: No Collective Phrase Detection

**Problem:**
- "all social platforms" is a **collective phrase** meaning multiple nodes
- The system doesn't detect collective phrases like:
  - "all social platforms" → instagram, twitter, linkedin, facebook
  - "all social media" → instagram, twitter, linkedin, facebook
  - "social platforms" → instagram, twitter, linkedin, facebook

**Current Logic:**
- Only checks for individual node names
- Doesn't handle collective phrases

### Issue 3: Keyword Extraction May Not Find "social platforms"

**Location:** `extractKeywordsFromPrompt()` (line 1499)

**Problem:**
- Checks if keyword matches prompt using word boundary regex
- "social platforms" might not match individual node keywords
- Individual nodes have keywords like "instagram", "twitter", "linkedin"
- But "social platforms" is a collective phrase, not an individual keyword

## Why Variations Are "Hardcoded"

The variations appear "hardcoded" because:

1. **Filter removes social platforms:**
   - User says "all social platforms"
   - Filter doesn't find "instagram", "twitter", etc. in prompt
   - Filters them out
   - Only keeps: schedule, ai_chat_model (which ARE in prompt)

2. **AI defaults to common pattern:**
   - Without social platform nodes in required list
   - AI generates common pattern: Google Sheets + Gmail
   - This is NOT hardcoded - it's the AI's default behavior

3. **No enforcement for collective phrases:**
   - System doesn't detect "all social platforms"
   - Doesn't expand to individual nodes
   - Doesn't enforce them in variations

## Solution Needed

### Fix 1: Detect Collective Phrases

Add detection for:
- "all social platforms" → instagram, twitter, linkedin, facebook
- "all social media" → instagram, twitter, linkedin, facebook
- "social platforms" → instagram, twitter, linkedin, facebook

### Fix 2: Expand Collective Phrases to Individual Nodes

When "all social platforms" is detected:
- Extract ALL social platform nodes from registry
- Add them to required nodes list
- Don't filter them out

### Fix 3: Update Filter Logic

Modify `filterToExplicitlyMentionedNodes()` to:
- Check for collective phrases FIRST
- If collective phrase detected, expand to individual nodes
- Then apply filtering logic

## Summary

**Why variations are "hardcoded":**

1. ❌ **Filter is too strict** - Removes social platforms because "all social platforms" doesn't contain "instagram"
2. ❌ **No collective phrase detection** - Doesn't understand "all social platforms" means multiple nodes
3. ❌ **AI defaults to common pattern** - Without required nodes, AI generates Google Sheets + Gmail

**The variations are NOT actually hardcoded** - they're generated by AI, but the AI doesn't have social platform nodes in the required list because the filter removed them.
