## Pattern Generation System

### Explicit Patterns (Manual)
- Defined in `EXPLICIT_NODE_TYPE_PATTERNS` array
- Highest priority
- Used for critical/ambiguous nodes
- Override auto-generated patterns

### Auto-Generated Patterns (Dynamic)
- Generated by `generateAllNodePatterns()` from node schemas
- Extracts patterns from all metadata fields
- 5-10+ patterns per node automatically
- Merged with explicit patterns

### Pattern Matching Strategy
1. **Exact Match** (case-insensitive) - Fastest
2. **Pattern Match** (word-boundary regex) - Primary method
3. **Token Match** (whole tokens only) - Fallback
4. **Levenshtein Distance** - Last resort

---

## Word Boundary Protection

**Critical Feature:** All patterns use word boundaries (`\b`) to prevent false matches:
- ✅ `/\bai\b/i` matches "ai" as whole word only
- ✅ `/\bgmail\b/i` matches "gmail" as whole word only
- ❌ "gmail" will NOT match "ai" because "ai" is not a whole word in "gmail"

**Example:**
- Input: "gmail" → Matches `google_gmail` ✅
- Input: "ai" → Matches `ai_service` ✅
- Input: "gmail" → Does NOT match `ai_service` ✅ (word boundary protection)
